Project/Account Cancellation

This list describes the steps that you should follow when receiving official cancellation ticket/notification

  • Clean page queues including new images and new segments
  • Delete existing scheduled jobs
  • Delete remaining segments and files in New
  • Turn off all Auto-Jobs (Auto-Parser, Auto-Assignment, etc.)
  • Unassign the account/project in TPM and TM2
  • Add a note to Project Information tab in TM2 with the info referencing the ticket
  • Send a notification to the vendors and Vendor TAMs (MLG) with the notification
  • Download and archive TMX files with the translations to V:\ drive
  • Set the account/project as inactive in Project Configuration in TPM
  • Set the default contractor to None

Project/Account Reassignment

This list describes the steps that you should follow when reassigning a project/account

  • Make sure that the account/project is assigned in TEs name in TPM
  • Assign the account/project in TEs name in TM2 project information
  • Check scheduled jobs
  • Check auto-jobs
  • Update One Note info and copy ICM/COS info into your section or create a new one if there's none yet
  • Check if there are any chats/groups in Teams for the project to add the TE there

When reassigning an account to MLG, please follow these steps

  • Contact Alejandro via a ticket to let him know the account is being reassigned to MLG
  • Make sure One Note is updated and put it's screenshot in the ticket
  • Attach to the ticket: Glossary, Style Guide, Appendix A
  • Send the ticket to Contractor Manager (MLG) and notify of the transition
  • Once acknowledged by MLG, please assign the account in TPM to MLG
  • Send the ticket to Alejandro for reference and tracker update

Post-delivery WE distribution queues

Customers no longer have an assigned post-delivery (maintenance) Web Engineer. Tickets that are not in process have been moved into one of four communal distribution queues: Misalignment, Translation, Functionality, Other.

WE - Misalignment

  • Visual Element on a page is misaligned compared to origin

WE - Translation

  • Appendix A (Rule A/B/C/D scope of work revisions, MotionPortal URL entries)
  • Text and image activations
  • Content that has been translated but not showing translated on the page
  • Content that is needs WE assistance to enter the translation queue
  • Content optimizations (DTs, merges, breaks, blocks)
  • WebCATT content issues (encoding, code in queue, unwanted pages)
  • Broken images in translation queue
  • WebCATT project/page queues
  • Machine translation

WE - Functionality

  • Unexpected page behavior
  • On Site Search (OSS)
  • TranSend
  • Easylink
  • Speedbump/splash page
  • Global Site Map (GSM)
  • Content customization

WE - Other

  • Site outage or uptime and availability
  • Customer call requests, root cause analysis
  • Platform or technology setup questions, quotes, whitelisting
  • WebCATT inaccessible/TServer properties not found

Audits Process

This list describes the steps that you should follow when performing site audits per received tickets

Select Pages

  • Create a ticket and request SDM to provide Top 50 URLs by recent traffic extracted from Logaholic
  • In case no analytics available, please build the list of URLs to be reviewed by selecting the most visible pages (Homepage, Landing Pages, Promo Pages, pages with recent translation activity
  • From the list, provided by SDM, select top 20-30 URLs and validate that they are working. Skip those that don't work moving down to the next one.
  • Make sure your shortened list contains HP/LP/Promos as well
  • Check the URLs against Appendix A to make sure they are within Scope

Submit Audit Request by sending the ticket to MLG with the following

  •     1. List of URLs to review
  •     2. General word count on the URLs
  •     3. Most updated Glossary and Style Guide which can be found in TPM for that account
  •     4. Expected process to follow (include Audit Process for Vendors files)
  •     5. Translation Review Template to be filled out

Expect MLG to return the ticket to you with the following

  •     1. All reviewed URLs are listed out
  •     2. All findings are listed out
  •     3. All errors, discrepancies, faults, etc. are cataloged per MP Translation Template

Scorecard information

  • Load the reviewed URLs into the queue
  • Run update job to determine the word count of all the pages combined
  • Fill out the Scorecard information (customer name, word count, page count, etc)
  • For any technhical instances please direct child tickets to WE queues accordingly before proceeding sending the ticket to PTS to review

Audit Report Processing

  • Send the filled out report to PTS team (MN QA Issues) to review
  • Once the ticket is back to you, make sure that the template has all listed issues reviewed and marked accordingly before the final scores are calculated
  • PTS is to present and Incident Report to MP in case of a failing score, otherwise they provide their assessment of the audit results

Results and further steps

  • PTS provides their assessment of the audit results
  • Generate a report by site regardless of the score, providing visibility into the overall and linguistic scores summarizing problems found, corrective measures taken or required, brief background on the customer's feedbacks
  • Share the report with CSM, SDM and Head of Translation Department
